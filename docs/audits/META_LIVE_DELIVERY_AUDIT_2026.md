# Meta Live Delivery Audit 2026

Date: 2026-06-07

## Goal

Resolve the contradiction between local code audits and Meta Test Events UI signals:

- Code/browser probes showed `fbq` executing but no visible `facebook.com/tr` request.
- Meta Test Events UI reportedly showed `PageView` and `Subscribe`.

## Findings

### Actual transport endpoint

- No alternate transport endpoint was observed in automated browser probes.
- The only Meta network activity consistently observed was:
  - `https://connect.facebook.net/en_US/fbevents.js`
  - `https://connect.facebook.net/signals/config/...`
- No `facebook.com/tr` request was captured in the automated probes.

## Root Cause

The application code correctly initializes `fbq` and fires the standard Meta event calls, but automated browser probing does not show those events leaving the browser as a Meta pixel transport request. The `Subscribe` hit shown in Meta Test Events UI is not generated by any code path found in this repository, so it is likely coming from a different browser/session, a delayed event, or another source outside the audited codebase.

## Deduplication

- No event-id based deduplication path is present in the current Meta wrappers.
- Deduplication is therefore not the observed blocker for the missing transport in automated probes.

## Recommendation

- Recheck the live production site in a non-headless Chrome session with Meta Pixel Helper and the Network tab open.
- If Meta Test Events continues to show `PageView`/`Subscribe` while DevTools still shows no `facebook.com/tr`, treat the Test Events hits as external to this audited code path until a live browser capture proves otherwise.
